# Port Search — Memory bank (stable reference)

Single place for **long-lived** project facts. For a short "where we are now," see `CONTEXT_RECORD.md`.

## What this project is

- **Port Search** — a fast lookup app to find **ports and their port codes** (UN/LOCODE). Type a port name, code, or country and get matching ports.
- **Sibling app to PTS** (Precision Timing Schedule). Shares PTS's **design language** and stack on purpose.
- **Stack:** static **HTML + CSS + vanilla JS**, no build step. Data lives in a plain JS file (`airports-data.js`) that defines `window.PORTS`.
- **Canonical files** in repo root: `index.html`, `app.js`, `styles.css`, `airports-data.js`.
- **Generators (dev-only, gitignored deps):** `gen-airports.js` (builds `airports-data.js` from OurAirports CSV), `gen-icons.js` (builds the PWA PNG icons).

## Design system (carried over from PTS)

- **Colors** (CSS variables in `styles.css`):
  - `--header-red #B71C1C` (app bar, accents, focus rings, card borders)
  - `--accent-grey #424242` (secondary text/UI)
  - `--content-bg #F5F5F5`, `--card-bg #FFFFFF`, `--border-light #E0E0E0`
  - `--text-body rgba(0,0,0,0.87)`, `--text-muted #757575`
- **Patterns:** red app bar (min-height 56px) with a context **pill**; `main` capped at **720px**, centered, mobile-first; **white cards with red borders**, 8px radius; red focus borders on inputs; system font stack.
- **Layout:** full-height (`100dvh`) flex column — app bar, search box, results-meta, and the code **legend** stay pinned; only the results list scrolls.
- **Screen pattern:** single-page app using `<section class="screen">` blocks toggled with a `.hidden` class (same as PTS). Currently one screen (`#searchScreen`); add more sections as needed.
- **PWA:** installable — `manifest.webmanifest` + `apple-touch-icon.png` / `icon-192.png` / `icon-512.png`; inline SVG favicon ("PS" red tile).

## Data model

- **`airports-data.js`** defines `window.PORTS` = array of:
  `{ code, iata, name, city, country, region, type }`
  - `code` — ICAO code (4-letter), e.g. `YSSY`.
  - `iata` — IATA code (3-letter), e.g. `SYD`. **Display prioritises IATA** (primary), ICAO shown as the secondary line.
  - `city` — municipality / city served.
  - `region` — continent name (Asia, Europe, Africa, North America, South America, Oceania, Antarctica).
  - `type` — `"Airport"` (also `"Heliport"`, `"Seaplane base"`, `"Balloonport"`).
- **Data = every airport worldwide that has an IATA code (~9,055 rows).**
  - **Source (public domain):** OurAirports — https://ourairports.com/data/
  - Generated by `gen-airports.js` from `airports.csv` + `countries.csv`; rows sorted major-airports-first, `[Duplicate]`-flagged and `closed` entries excluded.

## Search behaviour (`app.js`)

- **Tokenized search:** query is lowercased and split on whitespace; an entry matches only if **every** token is found in the combined `code + iata + name + city + country + region` haystack.
- **Highlighting:** matched substrings wrapped in `<mark>`; all rendered text is HTML-escaped first (XSS-safe).
- **Result cap:** `MAX_RESULTS = 100`; meta line shows "Showing 100 of N" when capped.
